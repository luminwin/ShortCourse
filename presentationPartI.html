Brief Overview

randomForestSRC is a fast OpenMP and memory efficient package for fitting random forests (RF) for univariate, multivariate, unsupervised, survival, competing risks, class imbalanced classification and quantile regression.

Brief Overview

A random forest (RF) consists of a collection of tree structured predictors {\(h({\bf x},{{\bf\Theta}_b}), b= 1,2,\dots\)} where {\({{\bf\Theta}_b}\)} are independent identically distributed random vectors. The RF predictor is the ensemble \[\hat{F}_{\text{ens}}({\bf x})=\frac{1}{B}\sum_{b=1}^B h({\bf x},{{\bf\Theta}_b})\]

Brief Overview

rfsrc(formula, data, ntree, mtry, nodesize)

Grow (train) your RF through rfsrc
Specify your model in formula
Provide your data frame in data
Tune your model via ntree, mtry, nodesize

Quick Start

Tip

The last line(s) will show the cross validated prediction performance

> 14                      (OOB) R squared: 0.7745644
> 15                     (OOB) Error rate: 245.3191
  • Model performance is displayed in the last two lines of the output in terms of out-of-bag (OOB) prediction error. In the above regression model, this is evaluated as the cross-validated (leave-one-out bootstrap) mean squared error (MSE) estimated via the out-of-bag data shown in line 15
  • Since MSE is scale dependent, standardized MSE, defined as the MSE divided by the variance of the outcome, is used and converted to R squared or the percent of variance explained by a random forest model which has an intuitive and universal interpretation, shown in line 14

What is OOB prediction error?

  • Since each tree is grown from a random subset of the data, the package returns both out-of-sample and in-sample predicted values from the forest, where the former are calculated using the hold out data for each tree ($predicted.oob), and the latter are from the data used to train the tree ($predicted)
  • The out-of-sample values $predicted.oob should be used for inference on the training data since they are cross-validated and will not over-fit the data. See the Forest Weights, In-Bag (IB) and Out-of-Bag (OOB) Ensembles vignette for a more formal description of IB and OOB and how they are used to define the ensemble 
> 14                      (OOB) R squared: 0.7745644
> 15                     (OOB) Error rate: 245.3191

we can get the error rate (mean-squared error) directly from the OOB ensemble by comparing the response to the OOB predictor below

print(mean((airq.obj$yvar - airq.obj$predicted.oob)^2))
> [1] 245.3191

Data

Data structures allowed

We only allow data.frame format and you have to use the as.data.frame() command to convert matrix, tibble, etc

Clean up your data

Choose your variables in formula and grow a tree

o <- rfsrc(y ~ a + z, data= dta, ntree = 1)

your outcome(s) will be saved in o$yvar and your predictors are in o$xvar from dta without missing values

Impute your data

To impute your data, use the following commands

o <- impute(y ~ ., data = dta, nimpute = 3) # for supervised problem
o <- impute(data = dta) # for unsupervised problem

Tip: factors or characters

Unlike lm(), glm(), or machine learning packages like ranger, rfsrc() does not require contrast/dummy coding

The mean-squared error (mse) splitting rule

For a scalar continuous outcome value \(Y\), suppose the node \(t\) uses the ordered or numeric feature \(X\) to split with the full sample size \(n\) in the root node. Letting \(Y_1,\ldots,Y_n\) denote the scalar outcomes in \(t\), the mean-squared error (mse) split-statistic for a proposed split \(s\) is \[ D(s,t) =\frac{1}{n}\sum_{i\in t_L}(Y_i-\overline{Y}_{t_L})^2 + \frac{1}{n}\sum_{i\in t_R}(Y_i-\overline{Y}_{t_R})^2 \] where \(t_L=\{X_i\le s\}\) and \(t_R=\{X_i>s\}\) are the left and right daughter nodes and \(\overline{Y}_{t_L}\) and \(\overline{Y}_{t_R}\) are the sample means for \(t_L\) and \(t_R\) respectively. The best split for \(X\) is the split-point \(s\) minimizing \(D(s,t)\) [Chapter 8.4 in 2]

Regression example: Iowa housing

The quantile.regr and la.quantile.regr split

  • quantile.regr refers to quantile splitting using quantile loss (also called pinball loss) and is defined using the check-loss function. It requires specifying the target quantiles

  • la.quantile.regr refers to local adaptive quantile regression splitting (the default action for quantreg)

Zhang H., Zimmerman J., Nettleton D. and Nordman D.J. (2019). Random forest prediction intervals. The American Statistician. 4:1-5.
